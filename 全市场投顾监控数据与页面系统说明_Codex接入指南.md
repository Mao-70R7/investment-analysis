## 5. 核心计算口径

### 5.1 策略唯一性和渠道口径

策略以 `统一策略ID` 为主键。这个字段避免不同渠道或不同 App 中策略代码冲突。做跨表 join 时优先使用 `统一策略ID`，不要只用策略名称。

渠道统计时，页面会展示业务名称，例如“天天基金/投顾”“广发基金”。底层表会保留 `渠道ID` 和 `渠道策略ID`，用于追溯原始采集文件。

### 5.2 当前持仓口径

当前持仓来源分两类：

1. 渠道直接披露持仓：优先使用 `策略当前持仓`。
2. 推算持仓：如果直接持仓缺权重、日期滞后，或最新调仓更靠后，则用 `策略当前持仓推算补齐`。

推算逻辑：

- 从最近一次可用调仓后的基金权重出发。
- 用基金净值和分红复权更新每只基金的相对权重。
- 归一化后生成当前基金权重。
- 在 `最新持仓推算稽核策略汇总` 和 `最新持仓推算稽核基金明细` 中记录推算来源、置信度、差异和稽核结论。

因此页面上的“最新持仓日”可能来自直接披露日期，也可能来自推算日期。字段 `分类依据` 会说明持仓来源。

### 5.3 基金资产类型、行业和主题分类

基金分类不是简单的一只基金只能属于一个类型。系统支持基金在页面分析中拆分为多种暴露，例如一只基金可按业务规则拆成“金融 70% + 地产 30%”。

分类来源优先级：

1. `基金标准分类字典` 中的标准分类、主题标签、市场地域、主动被动、QDII、ETF/指数等标签。
2. 渠道披露的基金类型、分组名称、基金名称。
3. 规则兜底：根据基金名称关键词、跟踪指数、分组名称识别股票、债券、货币、混合、QDII、商品、行业主题等。

页面展示中的资产大类和行业分析会使用分类拆分后的暴露权重，而不是简单把基金硬塞到单一分类。调仓分析中“哪些基金导致该资产/行业比例变化”会落到基金级明细，方便追溯。

### 5.4 风险等级口径

页面主风险等级是系统测算字段，不直接等同渠道披露风险等级。测算逻辑：

- 计算权益暴露档：来自当前持仓中的权益、偏股、股票方向基金权重。
- 计算波动档：来自策略回放或披露收益曲线的年化波动率。
- 计算回撤档：来自策略最大回撤。
- 最终风险等级取三项中最高风险档，即 `max(权益暴露档, 波动档, 回撤档)`。

渠道披露风险等级保留为 `披露风险等级`，用于对照。

### 5.5 策略业务分类和研报分类

页面同时保留多个分类维度：

| 字段 | 含义 |
|---|---|
| `业务分类` | 面向投顾业务运营的互斥主分类，例如目标盈系列、固收+、权益增强等。 |
| `研报产品类型` | 面向研报和竞品分析的产品类型口径。 |
| `研报股票子类型` | 权益类或行业主题类策略的子分类。 |
| `业务组合分类` | 披露策略类型和系统业务分类的组合标签。 |
| `市场地域` | 国内、海外/全球、国内+海外。 |
| `主动被动` | 主动为主、指数/被动为主、主动被动混合。 |
| `策略实现标签` | 由底层工具属性生成的多标签，例如主动基金组合、指数工具组合、QDII/海外工具。 |

这些分类字段的解释会写入 `basic_summary.js` 的 `fieldDictionary`，页面上字段弹窗会展示具体口径。

### 5.6 调仓事件去重和明细口径

底层原始调仓记录可能包含重复披露、同日多载荷、同一 App 滚动采集重复写入等情况。系统处理方式：

- 原始数据不硬删除，保留可追溯性。
- 页面和质量分析按业务键折叠：`统一策略ID + 调仓日期 + 调仓标题 + 调仓原因 + 调仓前后仓位结构`。
- 同一个事件内的基金明细按基金代码、基金名称、动作和权重变化汇总。
- 如果基金代码缺失，先用基金名称和标准字典补码；补不到时保留原始名称并标记解析状态。

页面“近期调仓明细”和“历史调仓”显示的是折叠后的业务事件，不应直接用 `策略调仓明细` 的原始行数判断事件数量。

### 5.7 调仓后收益和贡献曲线

调仓后收益分析优先使用 `调仓质量事件分析` 和 `调仓质量基金明细`。

事件级逻辑：

- 区间起点：调仓日期或调仓后首个可交易净值日。
- 区间终点：下一次调仓日前一交易日；如果没有下一次调仓，则用当前可用基金净值最新日。
- 有效性：基金净值覆盖、调仓前后权重、基金代码解析状态等都满足条件才纳入可评估事件。

基金级贡献逻辑：

- `基金区间收益率_百分比` 来自基金净值在调仓区间内的收益。
- 贡献大致按 `调后权重 × 基金区间收益率` 估算，并结合调前权重判断买入、卖出、增配、减配、持有。
- 对同一基金早期代码或份额缺失的情况，系统会用同基金可识别份额做兜底，但会在质量表中保留说明。

### 5.8 策略业绩回放

策略标准业绩净值使用统一回放算法：

1. 读取历史调仓后的基金权重。
2. 用基金日度净值和分红送配做复权。
3. 生成费前和费后单位净值。
4. 和官方披露净值做同区间对比。
5. 输出偏差、最大回撤、波动率、夏普、年化收益等指标。

页面默认优先展示贴近 App 的披露业绩；自建回放用于校验、补充、同口径比较和质量分析。

### 5.9 基准和市场指数

基准来源按优先级使用：

1. 渠道或 App 直接披露的日度基准收益。
2. 从业绩基准文本中解析指数和权重，用 `指数日度行情` 推算。
3. 无法解析时，使用可明确映射的通用指数作为参照，例如沪深300。

字段 `基准可用状态`、`基准权益权重`、`基准债券权重`、`基准货币权重` 用于判断基准解析质量。

## 8. 生成和刷新链路

统一增量更新入口：

```bat
E:\synctingData\全市场投顾监控\运行统一增量更新并刷新报表.bat
```

底层主入口：

```bat
E:\synctingData\全市场投顾监控\run_incremental_update_and_refresh_report.bat
```

典型流程：

1. 天天投顾增量采集：发现新策略、新调仓、最新业绩和持仓。
2. 广发基金更新：采集策略、持仓、调仓和业绩曲线。
3. 统一后处理：入库、质量检查、基金净值补齐、策略回放、调仓质量分析。
4. 页面导出：`scripts\export_basic_data_pages.py` 生成 `site\basic_data`。
5. 洞察包构建：`scripts\apply_field_renames_and_build_insights.js` 生成字段说明、基金详情包、仓位和调仓洞察包。
6. 部署校验：`scripts\write_analysis_platform_deploy_manifest.py` 输出 `deployment_manifest.json`。
7. 同步分享目录：把完整 `basic_data` 和部署清单同步到 `E:\synctingDataToWork\全市场投顾分析平台`。

常见开关：

| 变量 | 作用 |
|---|---|
| `RUN_TTFUND_UPDATE=0` | 跳过天天投顾采集，只做后处理和报表刷新。 |
| `RUN_GFFUNDS_UPDATE=0` | 跳过广发采集。 |
| `GFFUNDS_RUN_FULL_COLLECT=0` | 广发不跑完整公开采集，只做较轻量更新。 |

这些开关适合排障或只刷新报表时使用。正式例行更新建议使用默认值。

## 10. 自定义分析注意事项

1. 不要直接改写 `analysis_zh_current.sqlite`。自定义分析应只读连接，或复制一份数据库再实验。
2. 百分数字段一般已经是百分比单位，例如 `5.23` 表示 `5.23%`，不是 `0.0523`。
3. 日期字段多为文本，应按 `YYYY-MM-DD` 排序或转日期。
4. 策略名称不是主键，跨表关联用 `统一策略ID`。
5. 基金名称不是主键，跨表关联优先用 `基金代码`。
6. 调仓明细原始行数不等于业务事件数，页面和质量分析会做折叠去重。
7. 页面数据包很大，Codex 分析时不要一次加载所有详情文件；先从 `basic_summary.js` 或 SQLite 筛选目标，再读取少量详情。
8. `fieldDictionary` 是页面字段口径的第一优先解释来源。若口径说明不够清楚，应回到加工脚本补充字段说明，而不是只改页面文案。
9. 如果双击 HTML 打开遇到浏览器本地文件限制，建议在结果目录启动一个静态服务器，例如用 Python `python -m http.server 8800`，再访问 `http://127.0.0.1:8800/basic_data/index.html`。
10. Syncthing 同步时可能出现 `~syncthing~*.tmp` 或 `*.sync-conflict-*`，这些不是正式数据文件。

## 12. 页面字段说明维护

页面中字段说明来自两部分：

1. `basic_summary.js` 内的 `fieldDictionary`。
2. 页面脚本中对复杂图表、热力图、弹窗的局部说明。

维护原则：

- 能解释为具体公式的字段，必须写公式。
- 能说明数据表来源的字段，要写具体表名和字段名。
- 规则分类字段要写优先级和兜底逻辑。
- 不能只写“该字段来自当前分析库，页面仅做业务化展示”这类泛泛说明。

例如：

| 字段 | 有效说明应包含 |
|---|---|
| `风险等级` | 权益暴露档、波动档、回撤档如何计算，最终为何取最高档。 |
| `权益基金权重` | 来自当前持仓或推算持仓，按基金分类字典归入权益/偏股/股票方向后求和。 |
| `调仓后收益` | 调仓日期到下一调仓前一交易日或最新净值日的基金/组合收益。 |
| `调仓贡献` | 基金调后权重乘以调仓后区间收益，并按基金或分类汇总。 |
| `市场Top5平均收益` | 同分类内按所选收益指标排序，取前 5 个可计算策略的平均值。 |
